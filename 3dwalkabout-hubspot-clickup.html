Acceptance checks

What done should mean

Inputs needed

  • The HubSpot deal stage or stages that should create a ClickUp task.
  • The ClickUp list, custom fields, owner rules, and task naming pattern.
  • The HubSpot fields that need ClickUp task ID, status, owner, or last-sync notes.
  • Three to five representative deals, including one duplicate-risk case and one missing-field case.

Done means

  • Approved HubSpot deals create one matching ClickUp task and do not duplicate on reruns.
  • Existing tasks update instead of recreating when deal properties change.
  • ClickUp task ID and selected status fields write back to HubSpot.
  • Failures produce a clear review log instead of silently stopping the scenario.
