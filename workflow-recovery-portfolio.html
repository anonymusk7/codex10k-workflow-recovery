Reusable deliverable types

Field mapsSource, target, validation, dedupe, and ownership notes for messy handoffs.
Scenario registersTrigger, action, owner, failure path, and recovery note for each workflow.
Test fixturesDummy rows, sample payloads, and expected outputs before live access is needed.
RunbooksConcise handoff docs so a team can rerun, debug, or safely extend the workflow.
Data boundary: the first pass can use redacted screenshots, dummy rows, synthetic payloads, exports, or a least-privilege sandbox invite. Do not send passwords, API keys, card data, production secrets, private customer records, or regulated records by ordinary email.
